So a couple things to note here. If you've never worked with a Raspberry Pi before, getting the starter kit is a safe choice. It comes with everything you need, and there more than enough resources to help you trouble shoot.

For the monitor, 24" felt like a good size for me. Any bigger and you would need to buy a bigger sheet of 2 way mirror, a larger IR frame, and more lights(?)! In short, bigger monitor = more money (duh). You could also try to get a touch screen monitor to remove any need to buy an IR frame. Personally, I had a hard time finding a monitor that supported Raspbian OS. I actually initially ordered a touch screen monitor. I couldn't get it to work, so I just ended up shipping it back.

I actually bought this IR touch screen frame off of Alibaba. All the 24" ones on Amazon were easily over $300! I didn't provide a link because the products seem to change URLs quite often. You should be able to find what you need simply by searching {your monitor size} IR Touch Screen Frame.

There's not much additional insight I can offer on this. I chose 3mm over a thinner mirror because I wanted it to sit more flush with the monitor frame. Also, any thinner, and I feel like the thing was going to break on me.

Step 2. Setting Up the Pi

The instructions for this step should be provided with your starter kit order. In the case that it's not, here's a quick rundown.

  1. Format your MicroSD card (Raspberry Pi 3 takes a MicroSD, others take a normal SD card, I think).
    • Again the starter kit should come with everything you need, including a microSD card reader. In the event that you don't have a reader, you'll want something like this.
    • Formatting instructions on Windows/Mac found here.
  2. Download and extract the files from the NOOBS zip file.
    • File found here.
  3. Copy the extracted files over to the microSD card.
  4. Remove the SD card, and insert it to your Raspberry Pi 3.
  5. Power on the Raspberry Pi. You should now be sent to a set-up screen.
    • If your Pi isn't using the entirety of your monitor, go ahead and head over to /boot/config.txt and uncomment the line that says disable_overscan=1. Now, reboot your Pi by typing reboot in the terminal, or just hit restart. Your Pi should now be in full screen!

Step 3. Installing Node on the Pi

Instructions for this step are taken from DigitalOcean's guide found here. Here's a quick run-down though.

From the terminal, type the following:

  1. $ sudo apt-get update
  2. $ sudo apt-get install nodejs
  3. $ sudo apt-get install npm
  4. $ npm -v and $node -v to verify that at least you get 2.15.9 and 4.6.0 respectively (those are my numbers, but I haven't updated in a while).

Step 4. Download the Source Code && Dependencies

We're going to use MichMich's Magic Mirror source code, found here, because it seems like the most popular one, and it's very modular. Of course, at this point, you can just be finished and go ahead to create your own software for this. But what's the point of reinventing the entire wheel? Let's just upgrade our tires.

  1. In a directory of your choice, go ahead and type $ git clone https://github.com/MichMich/MagicMirror.git to copy the project files. You can also fork your own copy of this. Change the clone link, of course.
  2. cd in to the cloned folder and move config/config.js.sample to config/config.js
  3. In the project root, type $ npm install.

Step 4. Run it!

We're almost there! If everything went smoothly, you shouldn't of had scary looking errors during this download process

  1. Go ahead and type $ npm start to start up the mirror. To quit out of it hit CTRL + Q.

And there you have it! The software should all be working now. If you want to start the program remotely from another computer, check out the next step.

Step 5. Setting up SSH (optional)

More likely than not, you'll probably want to never plug in a mouse and keyboard to change some code for this mirror after you've mounted it and everything. This is where being able to ssh will come in handy.

  1. Type $ sudo raspi-config in the terminal.
  2. Select Interfacing Options.
  3. Navigate to and select SSH.
  4. Choode Yes, select Ok, and then hit Finish.
  5. Test your connection by typing something like $ ssh pi@{your IP} on your local machine (assuming macOS/Linux). Replace that IP address with whatever comes out from typing $ hostname -I on the Raspberry Pi.
